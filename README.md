# Active Directory Security & Detection Lab

## Objective
The objective of this project was to design and deploy a full enterprise network environment to understand corporate infrastructure security, identity management, and defensive telemetry auditing. By building a Windows Active Directory domain (`mydfir.local`), configuring endpoint logging tools, and using simulation frameworks like Atomic Red Team, I established a complete pipeline to generate, forward, and analyze enterprise security event logs within a centralized Splunk SIEM instance.

### Skills Learned
- **Enterprise Network Architecture:** Designing host environments, managing IP subnets, and configuring client-server domain topologies.
- **Active Directory Administration:** Understanding domain environments, user/computer objects, and central policy management.
- **Log Pipeline Engineering:** Deploying and configuring Splunk Universal Forwarders to ship Sysmon and Windows Event logs securely to a centralized indexer.
- **Emulation of Adversary TTPs:** Utilizing Atomic Red Team to simulate realistic attacks mapped directly to the MITRE ATT&CK matrix.

### Tools & Technologies Used
- **Active Directory / Windows Server:** Domain controller regulating identity and access management policies.
- **Splunk Enterprise & Universal Forwarder:** Log collection, transport, parsing, and correlation.
- **Sysmon (System Monitor):** Implemented on endpoints for deep structural insight into system activities.
- **Atomic Red Team:** Automated framework used for launching defensive test cases against the domain.
- **Kali Linux:** Dedicated attack platform for target infrastructure reconnaissance and testing.

## Lab Implementation & Configuration Steps

### Step 1: Active Directory Domain Deployment
- Provisioned a **Windows Server** virtual machine acting as the Primary Domain Controller for the `mydfir.local` domain.
- Configured core networking services including static IP assignment (`192.168.10.7`) and active DNS routing.
- Joined a **Windows 10 client machine** to the domain to act as the primary target workstation for simulation telemetry.

### Step 2: Endpoint Telemetry Setup (Sysmon & Universal Forwarder)
- Installed SwiftOnSecurity's optimized **Sysmon configuration** on both the Domain Controller and Windows 10 workstation to track process executions, network connections, and critical system alterations.
- Configured the **Splunk Universal Forwarder** on all endpoints, adjusting the `inputs.conf` file to capture and forward `WinEventLog:Security` and `Microsoft-Windows-Sysmon/Operational` telemetry channels to the main Splunk Indexer (`192.168.10.10`).

### Step 3: Adversarial Simulation with Atomic Red Team
- Installed the **Atomic Red Team** framework on the Windows 10 workstation to safely emulate techniques aligned with the MITRE ATT&CK matrix (such as credential dumping, discovery commands, or persistence mechanisms).
